# 手動検証チェックリスト

## 概要
このドキュメントは、Start Board Dashboard拡張機能の手動検証手順を定義します。すべての主要機能が正しく動作することを確認するために使用します。

## 前提条件
- VSCodeがインストールされていること
- Start Board Dashboard拡張機能が開発モードでインストールされていること
- 検証用のテストプロジェクト（フォルダ/ワークスペース)が準備されていること

## 検証手順

### 2. プロジェクト追加コマンド実行
**要件:** 4.2
**目的:** プロジェクト追加コマンドを実行後、リストが更新されることを確認する

**手順:**
1. VSCodeでテスト用のフォルダまたはワークスペースを開く
2. コマンドパレット（Ctrl+Shift+P / Cmd+Shift+P）を開く
3. `Start Board: 現在のプロジェクトを追加` コマンドを実行する
4. 成功メッセージが表示されることを確認する
5. 空のウィンドウを開いてダッシュボードを表示する
6. プロジェクトリストに追加したプロジェクトが表示されることを確認する

**期待される結果:**

- [x] 既に追加済みのプロジェクトを再度追加しようとすると警告メッセージが表示される

### 4. プロジェクト削除機能
**要件:** 6.3

### 5. アスキーアート設定変更
**要件:** 2.6
**目的:** 設定変更後、アスキーアートが更新されることを確認する

**手順:**
1. ダッシュボードを表示する
2. VSCodeの設定画面を開く（File > Preferences > Settings / Code > Preferences > Settings）
3. `startBoard` で検索する
4. 以下の設定項目を変更する:
   - `Start Board: Ascii Art Text` - テキストを変更（例: "Hello" → "Test"）
   - `Start Board: Ascii Art Font Family` - フォントを変更（例: "monospace" → "courier"）
   - `Start Board: Ascii Art Font Size` - フォントサイズを変更（例: 14 → 20）
   - `Start Board: Ascii Art Line Height` - ライン幅を変更（例: 1.2 → 1.5）
5. ダッシュボードに戻り、変更が即座に反映されることを確認する

**期待される結果:**
- [x] テキスト変更が即座にアスキーアートに反映される
- [x] フォントファミリー変更が即座に反映される
- [x] フォントサイズ変更が即座に反映される
- [x] ライン幅変更が即座に反映される
- [x] 設定画面を閉じた後もダッシュボードの表示が維持される
