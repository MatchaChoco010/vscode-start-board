# Requirements Document

## Project Description (Input)
プロジェクトをアルファベット順に並べるように変更したい。

## Introduction
VSCode Start Board拡張機能において、プロジェクトリストをアルファベット順に表示する機能を追加します。現在のプロジェクトリストは登録順に表示されていますが、プロジェクト数が増加すると目的のプロジェクトを見つけにくくなります。アルファベット順のソートにより、プロジェクトの検索性と一覧性を向上させます。

## Requirements

### Requirement 1: プロジェクトリストのアルファベット順表示
**Objective:** ユーザーとして、プロジェクトリストをアルファベット順に表示したい。それにより、目的のプロジェクトを素早く見つけることができる。

#### Acceptance Criteria
1. When ダッシュボードが表示される、the システムはプロジェクトリストをプロジェクト名のアルファベット順（昇順）でソートして表示する
2. When プロジェクト名が日本語を含む、the システムはUnicodeコードポイント順でソートする
3. When プロジェクト名の大文字小文字が異なる（例: "Project" と "project"）、the システムは大文字小文字を区別せずにソートする
4. When 新しいプロジェクトが追加される、the システムはリストを再ソートして適切な位置に挿入する
5. The システムはプロジェクト名のみでソート順を決定し、パス情報は使用しない

### Requirement 2: ソート状態の永続化
**Objective:** ユーザーとして、VSCodeを再起動してもソート順が維持されることを期待する。それにより、一貫したユーザー体験を得られる。

#### Acceptance Criteria
1. When プロジェクトリストがソートされる、the システムはソート済みの順序をglobalStateに保存する
2. When VSCodeが再起動される、the システムは保存されたソート順でプロジェクトリストを復元する
3. When プロジェクトが削除される、the システムはソート順を維持したまま該当プロジェクトを削除する

### Requirement 3: 既存データの互換性
**Objective:** 開発者として、既存のプロジェクトデータとの互換性を維持したい。それにより、ユーザーのデータが失われないことを保証する。

#### Acceptance Criteria
1. When 既存のプロジェクトデータ（ソート前のデータ）が読み込まれる、the システムは既存データを自動的にアルファベット順にソートして表示する
2. If プロジェクトデータの読み込みに失敗する、then the システムはエラーをログに記録し、空のプロジェクトリストを表示する
3. The システムは既存の`Project`型定義と互換性を維持する

### Requirement 4: パフォーマンス
**Objective:** ユーザーとして、大量のプロジェクトがあってもソートが高速に実行されることを期待する。それにより、スムーズなユーザー体験を得られる。

#### Acceptance Criteria
1. When プロジェクトリストが100件以下の場合、the システムはソート処理を10ミリ秒以内に完了する
2. When プロジェクトリストが1000件の場合、the システムはソート処理を100ミリ秒以内に完了する
3. The システムはソートアルゴリズムとして効率的な比較ベースのソート（時間計算量O(n log n)）を使用する
